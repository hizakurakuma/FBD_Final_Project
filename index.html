一、摘要

這裡放 150–300 字左右的摘要。建議內容包括：研究動機、資料來源、主要方法、核心發現。例如：單人月卡是否存在角色固定效果、Bilibili 評論中的有效熱度是否比原始熱度更能解釋流水等。

研究對象與玩家語境說明（可直接引用或改寫）
《戀與深空》是一款結合戀愛互動、角色養成與限定抽卡活動的女性向遊戲。對玩家而言，每次卡池不只是新內容上線，也是角色魅力、劇情張力、卡面設計與課金決策同時發生作用的消費事件。因此，卡池開始前的社群討論熱度、評論內容結構與玩家情緒反應，具有作為卡池流水前置信號的研究價值。

二、研究問題

研究問題 1

不同池種（moon / sun / return / roulette）是否對流水有系統性影響？

研究問題 2

單人月卡中，不同男主是否存在顯著的角色固定效果？

研究問題 3

新男主登場後三個月內，是否存在新鮮感效應？

研究問題 4

Bilibili 主 PV 評論中的有效熱度與評論結構，是否能額外解釋卡池流水？

三、遊戲背景與名詞解釋

《戀與深空》是一款以角色互動、劇情推進與抽卡養成為核心的女性向遊戲。對不熟悉遊戲機制的讀者而言，本報告中的「卡池」、「月卡」、「日卡」、「單人池」、「混池」等名詞可能較陌生，因此先在此說明研究中會用到的基本概念。

卡池（Pool / Banner）

指玩家在特定活動期間內可抽取限定卡面的抽卡池。不同卡池有不同的角色配置、卡面規格與活動包裝，因此通常會對玩家課金意願造成不同影響。

PV（Promotional Video）

指官方在卡池開始前發布的宣傳影片。本研究使用主 PV 發布後至卡池開始前的社群評論資料，作為卡池預熱聲量的觀察窗口。

月卡（moon）

月卡是《戀與深空》的兩種卡牌種類之一 (另一種為日卡) ，屬於基本卡牌，通常包含互動卡面及相關劇情。月卡各自獨立，不需要倚仗其他卡片才有特殊效果。《戀與深空》活動的限定卡池多屬於月卡卡池 (簡稱月卡池)。

本報告中以 moon 表示月卡池。

日卡（sun）

日卡是《戀與深空》的兩種卡牌種類之一，屬於特殊卡牌。有別於月卡各自獨立，日卡需要抽取同一個主題下兩張成對的卡才能成套。日卡除了包含互動卡面及更龐大的敘事劇情外，對於玩家的戰鬥體驗更有決定性提升。因此日卡卡池 (簡稱日卡池) 的抽取強度通常遠大於月卡池，一位男主一年內最多推出一套新日卡

本報告中以 sun 表示日卡池。

由於日卡池的抽卡結構與玩家預期，和月卡池有所差異，因此本報告將月卡池與日卡池分開進行分析。

單人池（single）

指卡池只包含單一角色。由於角色集中，較適合用來比較不同男主之間是否存在穩定的商業號召力差異，因此本文關於「角色固定效果」的分析以單人池為主。

混池（all）

指卡池同時包含多位角色。混池的吸引力可能來自多角色疊加，因此其抽卡邏輯與單人池不同，通常需要分開討論。

復刻池（return）

指既有卡池內容再次開放的活動。復刻池通常不屬於主要新內容排期，且可能與其他卡池同時存在，因此在收益分析中常需要額外控制其重疊影響。

四星轉盤卡（4-star_roulutte）

指四星轉盤形式的特殊活動池。這類池在研究期間內出現次數少、制度位置特殊，且可能與其他活動重疊，因此本文通常將其視為補充型活動而非主要排期池。

流水

在玩家社群語境中，「流水」通常指遊戲在某段期間內的營收表現。本研究以七麥數據提供的 iPhone 每日預估收益作為流水的代理指標，用於比較不同卡池活動的相對表現。

預熱期（Pre-launch window）

指主 PV 發布後到卡池正式開啟前的這段時間。本文將這段期間視為觀察玩家預期、討論熱度與情緒反應的重要窗口。

3.1 為何要分池種分析？

不同池種的抽卡機制、角色配置、玩家預期與營收規模差異都很大。如果把單人月卡、混池月卡、單人日卡與復刻池全部放進同一模型，容易把制度差異誤當成社群討論度或角色效果。因此本文會先依卡池類型切分樣本，再分別建模。

3.2 為何要控制重疊活動？

在《戀與深空》的活動排期中，部分復刻池與四星轉盤卡不一定占據主排期，可能與主要卡池同時存在。若不控制這些重疊活動，則某一活動期間觀察到的收益，可能其實同時受到其他活動影響。因此本文另外建構 overlap 相關控制變數，以降低估計偏誤。

四、資料清理與變數建構

4.1 主要資料表 schema

revenue_daily_clean

每日收益主表：日期、收益、log 收益、年/月/週、是否週末。

events_clean

活動主表：池種、單人/混池、復刻、角色、節日標籤、版本與新角色變數。

event_outcomes

每個活動對應的 D1 / D3 / D7 收益、abnormal revenue 與 overlap controls。

4.2 節日與版本標籤

  • is_birthday：角色生日池
  • is_special_holiday：情人節、白色情人節、五一、國慶、農曆新年等
  • is_anniversary：周年慶相關活動
  • is_new_version：版本更新、新男主實裝、開服事件

4.3 新男主新鮮期變數

本研究將秦徹首登場池 不設防禁區、夏以晝首登場池 觸痛訊號 作為新角色起始點，並將其後三個月內之單人池標記為 is_new_character = 1。

顯示 R 程式碼範例 
qinche_first_date <- events_clean %>%
  filter(event_name == "不设防禁区") %>%
  pull(event_start) %>%
  first()

xiayizhou_first_date <- events_clean %>%
  filter(event_name == "触痛讯号") %>%
  pull(event_start) %>%
  first()

events_clean <- events_clean %>%
  mutate(
    is_qinche_fresh_period =
      is_single &
      has_qinche &
      event_start >= qinche_first_date &
      event_start < qinche_first_date %m+% months(3),

    is_xiayizhou_fresh_period =
      is_single &
      has_xiayizhou &
      event_start >= xiayizhou_first_date &
      event_start < xiayizhou_first_date %m+% months(3),

    is_new_character =
      is_qinche_fresh_period | is_xiayizhou_fresh_period
  )

五、研究設計

5.1 樣本切法

主分析先聚焦於非復刻的主排期卡池，並視研究問題分成單人月卡、混池月卡與單人日卡進行分析，避免不同抽卡結構混合造成模型解釋偏誤。

5.2 事件級變數

  • 池種：moon / sun / return / roulette
  • 角色：沈星回、黎深、祁煜、秦徹、夏以晝
  • 新角色新鮮期：is_new_character
  • 節日/版本：生日、特殊節日、周年、版本更新
  • 重疊活動控制：overlap_days_d3 等

5.3 Bilibili 評論特徵

原始熱度

評論數、24 小時評論數、開池前總評論數、最後 12 小時增速。

去水熱度

去重使用者數、重複文本比例、儀式性留言比例、單一用戶集中度。

有效熱度

抽卡意圖比例、高情緒比例、實質評論比例、角色提及比例。

互動品質

平均點讚數、高點讚評論比例。

六、模型與檢定方法

6.1 角色效果檢定

先以單人月卡非復刻樣本為主，檢定不同男主是否存在顯著流水差異。可使用單因子 ANOVA、Kruskal-Wallis 檢定與 OLS。

顯示角色效果檢定程式碼 
moon_single_df <- event_outcomes %>%
  filter(is_moon, is_single, !is_return_dummy) %>%
  mutate(
    male_lead = case_when(
      has_shenxinghui ~ "沈星回",
      has_lishen ~ "黎深",
      has_qiyu ~ "祁煜",
      has_qinche ~ "秦徹",
      has_xiayizhou ~ "夏以晝",
      TRUE ~ "other"
    ),
    male_lead = factor(male_lead)
  )

anova_m1 <- aov(log_rev_d1 ~ male_lead, data = moon_single_df)
summary(anova_m1)
TukeyHSD(anova_m1)

6.2 事件級迴歸

以 log_rev_d1 或 log_rev_d3_sum 作為依變數，先控制角色、池種、節日與 overlap，再加入 Bilibili 評論特徵比較模型解釋力。

顯示 OLS 模型範例 
m2 <- lm(
  log_rev_d1 ~ male_lead + is_new_character +
    is_birthday + is_special_holiday + is_anniversary + is_new_version +
    overlap_days_d3 + overlap_return_rows_d3 + overlap_roulette_rows_d3,
  data = moon_single_df
)
summary(m2)

八、討論與限制

8.1 討論

這裡可以討論：角色固定效果是否顯著、秦徹與夏以晝是否存在短期新鮮感紅利、月卡與日卡是否因抽卡制度不同而呈現不同的收益結構，以及 Bilibili 評論中的原始熱度與有效熱度哪一種更接近卡池的實際消費轉化。

若你的結果顯示角色效果顯著，則可進一步主張：在分析社群討論度對流水的影響之前，必須先控制角色固定效果，否則很容易把角色本身的吸引力誤判為 PV 預熱聲量的影響。若有效熱度比原始熱度更穩定，也可以說明玩家社群中的「大量留言」未必等同於「高購買意圖」。

8.2 研究限制

  • 七麥收益為第三方估計值，並非官方實收，因此更適合用於相對比較而非絕對金額推估。
  • Bilibili 評論可能受到平台文化、接運留言、控評或刪評影響，因此需要以去水與有效熱度指標輔助辨識。
  • 部分事件變數如 public opinion 含研究者手動整理成分，因此較適合作為輔助控制，而非主識別來源。
  • 少數特殊池種如四星轉盤卡樣本數有限，推論應以探索性結果理解。
